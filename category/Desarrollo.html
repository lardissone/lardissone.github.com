Beanstalk, control de versiones 2.0

Tue 23 October 2007
By Leandro Ardissone

In Desarrollo.

tags: sitiosservicios


En la compañía usamos Basecamp (apesta, porque es de código cerrado y muchas cosas se podrían mejorar, pero es lo que hay) y generalmente debemos usar distintas aplicaciones externas que este no posee, como Tickspot para llevar control del tiempo destinado en cada tarea y otros.
Lo bueno de Basecamp es que posee una API que permite la integración de distintas aplicaciones externas, como es el caso de Tickspot y de la que paso a hablar en este momento.

Beanstalk es un servidor de Subversion (que es un servidor de control de versiones, para llevar el control del código fuente de nuestros programas) que se integra plenamente con Basecamp, Fogbugz y Lighthouse.
Es justamente lo que estabamos necesitando últimamente en la empresa, poder llevar control de nuestro código desde el mismo Basecamp.

Obviamente no es un Trac, pero bueno en la Web 2.0 no todas son rosas. Pero se ve tan liiiindo.
